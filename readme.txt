该项目在原生apache-ode项目基础上进行二次开发，主要开发目标为：
1.实现invoke partnerLink失败时，自动从策略服务器获取可替换的partnerLink，保证业务的正常执行
2.实现在定义bpel时可创建partnerLink为指定策略服务器地址的节点，执行时识别到该节点时，会动态从策略服务器取得策略，并动态决定业务拓扑
注：
由于ode项目编译的特殊性（需由jruby进行编译），故不在此workspace进行编码。通过前期实践，总结出工作流程如下可能比较合理（暂时未想出更加便捷的编码方式）。
1.在sublime中进行开发，然后上传到svn服务器
2.在服务器中通过脚本从svn拉取最新代码，并编译部署
3.测试
4.只有测试通过的阶段性成果，才会同步到此git中。
